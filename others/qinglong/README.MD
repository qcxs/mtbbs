# MT论坛 青龙自动签到助手

青龙面板专用的MT论坛多账号自动签到脚本，支持概率签到、Cookie缓存、消息提醒、异常通知，稳定无广告

## 功能亮点

- ✅ **多账号支持**：一次配置，批量自动签到

- 🎲 **智能概率签到**：自定义时间段内概率递增签到，避免固定时间打卡风控

- 🍪 **自动缓存Cookie**：一次登录长期有效，无需频繁输入账号密码

- 📩 **论坛新消息提醒**：自动检测未读消息并推送

- ⏰ **自定义通知时间**：仅在指定时段发送通知，不打扰

- 🚨 **异常自动处理**：Cookie过期自动清空，下次运行重新登录

- 📝 **完整日志记录**：执行状态一目了然，支持青龙面板通知推送

## 环境变量配置

青龙面板 → 环境变量 → 新建变量，**必填1个，选填2个**

### 1. 必选：账号配置（mtbbs_accounts）

格式为**JSON数组**，支持多账号

```JSON

[
    {"username":"账号1","password":"密码1"},
    {"username":"账号2","password":"密码2"}
]
```

### 2. 可选：消息提醒时间段（mtbbs_message_time）

仅在该时段发送青龙通知，格式：`开始小时-结束小时`，多时段用逗号分隔

```Plain Text

8-12,18-22
```

- 示例：`9-21` 代表每天9点~21点才发送通知

- 不配置则**全天不发送通知**

### 3. 可选：概率签到时间段（mtbbs_sign_time）

在该时段内**概率递增签到**，格式同上

```Plain Text

0-23
```

- 概率规则：越接近结束时间，签到概率越高

- 不配置则**不自动签到**

- 配置`0-23`为全天概率签到

2. 签到判断：当天已签到则跳过，未签到则按概率触发

3. Cookie失效：自动清空缓存，下次运行重新登录

4. 通知规则：仅在配置的时段内发送成功/失败/消息通知

## 常见问题

### 1. 提示登录失败

- 检查账号密码是否正确

- 确认MT论坛账号无异常

### 2. 频繁提示Cookie过期

- 手动登录论坛后，删除缓存文件或清空环境变量重新登录

- 不要在多处同时使用相同账号签到

### 3. 不发送通知

- 检查是否配置`mtbbs_message_time`变量

- 确认当前时间在通知时段内

- 检查青龙通知渠道是否正常
